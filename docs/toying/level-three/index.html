#2.3 💻 Booleans for coding Apps


How does boolean logic tie into coding? When a programmer is coding they write in a special language called a syntax. Every coding language can have different syntaxes, but ultimately they are just different ways of representing our logic models using code.

Rather than writing enormous combinations of ORs, NOTs, and other logical operators, software engineers conveniently created coding languages packed in with features that allow us to say what we are trying to do more easily. In fact, nearly every coding language has support for boolean logic built-in, which is an essential part of your toolkit as a coder.

You might have noticed that when given a NOT operator with multiple values, it is sometimes easier to think of it as simply a request for whatever is missing. This is an example of how we can build more complex rules from simplistic systems. Writing good software is about learning to create apps with complex capabilities without making the code itself overly complex or messy.

Like speaking any language (human or machine), learning how to better express yourself (by using clean code and good design practices) allows you to say more while writing less. Spending more time to think about your code from a design perspective will allow you to save countless hours dealing with mind-numbing bugs and other issues.

Many coding languages have the capabilities to compare and transform other kinds of data more advanced than booleans, which is very powerful. We can do things like evaluate math expressions, compare text, and more. Watch this video to see this in practice, and to see the relationship between the boolean logic and coding mental models.
